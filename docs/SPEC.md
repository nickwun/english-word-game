# 小学生英文单词游戏化学习网页 SPEC v1.1

## 1. Problem Statement（问题陈述）

目标用户是一个 **8 岁、小学三年级、接触英语不到一年、基础较薄弱但对英语有兴趣的孩子**。

孩子当前使用的教材为：

```text
闽教版小学英语三年级下册新版
```

该册教材包含：

```text
Unit 1 Our Animal Friends
Unit 2 My Clothes
Unit 3 Time
Unit 4 Healthy Eating
Review 1
Review 2
```

当前问题不是孩子完全不愿意学英语，而是：

1. 传统背单词方式比较枯燥；
2. 单纯做练习题容易产生挫败感；
3. 家长缺少一个简单方式查看孩子学到了哪里、哪些词还不会；
4. 孩子缺少一个稳定、轻量、有反馈的每日英语学习入口。

因此，本产品要解决的核心问题是：

**通过一个课本同步、轻游戏化的网页，让孩子在低压力、有反馈、像闯关一样的过程中，完成英文单词的“认识”和“听懂”训练，并逐步建立每天学习 5 个词的习惯。**

第一版不追求完整英语学习能力，而是聚焦：

- 看到英文单词，知道中文意思；
- 听到英文发音，能理解对应含义；
- 每天完成一个小关；
- 错词能被记录并复习；
- 家长能看到基础学习结果。

---

## 2. Proposed Solution（方案描述）

产品形式是一个：

```text
闽教版三年级下册英文单词轻游戏化学习网页
```

核心结构为：

```text
Unit 主线闯关
+
每关 5 个单词
+
学习 / 练习 / 挑战三段式
+
星级反馈
+
错词复习
+
额外挑战
+
Unit 徽章
+
轻量家长面板
```

一句话概括：

**孩子每天进入网页，完成一个 5 个词的小关；答题过程中获得即时反馈；答错词进入错词本；完成 Unit 主线和错词复习后算“主线完成”；完成补充词挑战后解锁 Unit 徽章。**

## 3. Content Scope（教材与词库范围）

### 3.1 教材范围

第一版基于：

```text
闽教版小学英语三年级下册新版
```

教材范围只覆盖：

```text
Unit 1 Our Animal Friends
Unit 2 My Clothes
Unit 3 Time
Unit 4 Healthy Eating
```

第一版不做：

- 三年级上册；
- 四年级内容；
- 其他版本教材；
- 额外课外拓展词库。

### 3.2 MVP 范围

第一版采用：

```text
数据先整理 Unit 1–4
功能先完整实现 Unit 1
```

也就是：

#### 数据层

先整理完整三年级下册 Unit 1–4 词库。

#### 功能层

第一轮只完整实现：

```text
Unit 1 Our Animal Friends
```

Unit 2–4 可以先显示为：

```text
即将开放
```

这样做的原因是：

**一个 Unit 做完整，比四个 Unit 做半成品更有价值。**

## 4. Vocabulary Priority（词汇优先级）

词汇分为两类。

### 4.1 A 类：课文重点词

来源：

```text
教材 Contents 目录中的 Key words
```

用途：

- 进入 Unit 主线关卡；
- 是孩子必须学习的词；
- 用于判断 Unit 主线是否完成；
- 每关约 5 个词。

A 类词是产品第一优先级。

### 4.2 B 类：补充词

来源：

```text
Words and Expressions / Word List 中出现，但不属于目录 Key words 的词
```

用途：

- 进入额外挑战关；
- 不阻塞 Unit 主线完成；
- 完成后可以获得特殊星星；
- 完成主线、错词复习和额外挑战后，可解锁 Unit 徽章。

B 类词是拓展挑战，不是压力任务。

## 6. Core Gameplay（核心玩法）

产品采用：

```text
Unit 主线闯关 + 每关选择题小游戏
```

学习路径：

```text
首页
→ Unit 列表
→ Unit 1
→ 主线关卡
→ 错词复习
→ 额外挑战
→ Unit 徽章
```

每个 Unit 有两条路径：

### 6.1 主线路径

```text
A 类重点词
→ 普通关卡
→ 错词复习
→ 主线完成
```

### 6.2 额外挑战路径

```text
B 类补充词
→ 额外挑战关
→ 特殊星星
→ Unit 徽章
```

额外挑战不阻塞主线进度。

## 7. Level Flow（每关流程）

每个小关固定分为三段。

### 7.1 学习阶段

展示本关 5 个单词。

每个单词包含：

- 英文单词；
- 中文意思；
- 图片 / 图标 / emoji；
- 发音按钮。

目标：

让孩子先建立：

```text
英文形状
+
中文意思
+
图片概念
+
英文发音
```

之间的连接。

### 7.2 练习阶段

低压力选择题。

题型主要包括：

```text
看英文，选中文
听发音，选中文
```

目标：

让孩子先熟悉，不急着挑战。

### 7.3 挑战阶段

混合选择题。

题型包括：

```text
看英文，选中文
听发音，选中文
听发音，选图片
看图片，选英文
```

目标：

检查孩子是否真正建立了“音、形、义”的连接。

## 8. Question Types（题型）

第一版支持四种选择题。

### 8.1 看英文，选中文

示例：

```text
bear

A. 熊
B. 马
C. 鸟
D. 熊猫
```

训练目标：

```text
认识单词
```

### 8.2 听发音，选中文

网页播放：

```text
bear
```

孩子选择：

```text
A. 熊
B. 马
C. 鸟
D. 熊猫
```

训练目标：

```text
听懂单词
```

### 8.3 听发音，选图片

网页播放单词发音，孩子从图片中选择。

训练目标：

```text
声音 → 含义
```

### 8.4 看图片，选英文

展示动物或物品图片，孩子选择英文单词。

训练目标：

```text
含义 → 英文
```

第一版不做拼写输入题。

## 9. Feedback System（答题反馈机制）

反馈采用：

```text
答对奖励
+
答错提示
+
二次机会
+
错词记录
```

### 9.1 答对时

系统反馈：

- 显示正确；
- 播放单词发音；
- 出现轻量动画；
- 增加星级进度。

示例反馈：

```text
答对啦！
bear，熊。
```

### 9.2 第一次答错时

系统不直接判死。

反馈：

- 给出提示；
- 保留本题；
- 让孩子再选一次。

示例：

```text
再想一想，听一遍发音。
```

### 9.3 第二次仍然答错时

系统反馈：

- 显示正确答案；
- 播放正确发音；
- 将该词加入错词本。

示例：

```text
正确答案是 bear，熊。
我们稍后再练一次。
```

这个机制的核心是：

**不让孩子乱点过关，也不让孩子因为一次答错就被打击。**

---

## 10. Reward System（奖励系统）

第一版奖励系统保持克制。

### 10.1 主线关卡奖励

每个主线关卡完成后给出：

```text
1–3 颗普通星星
```

星级依据：

- 正确率；
- 是否使用第二次机会；
- 挑战阶段表现。

### 10.2 额外挑战奖励

额外挑战关完成后给出：

```text
特殊星星
```

特殊星星用于区别普通关卡。

### 10.3 Unit 徽章

当一个 Unit 达到“完全掌握”状态后，解锁 Unit 徽章。

Unit 1 徽章建议：

```text
动物朋友达人
```

第一版只做徽章展示。

不做：

- 金币；
- 商店；
- 宠物；
- 装扮；
- 排行榜；
- 道具兑换。

## 11. Unit Completion Rules（Unit 完成规则）

每个 Unit 有两级完成状态。

### 11.1 主线完成

满足条件：

```text
完成该 Unit 所有 A 类重点词主线关卡
+
完成该 Unit 错词复习
```

显示状态：

```text
主线完成
```

含义：

孩子已完成本单元核心词汇学习闭环。

### 11.2 完全掌握

满足条件：

```text
完成该 Unit 所有 A 类重点词主线关卡
+
完成该 Unit 错词复习
+
完成该 Unit B 类补充词额外挑战
```

显示状态：

```text
完全掌握
```

同时解锁：

```text
Unit 徽章
```

这个规则非常重要：

**主线词是必须完成，补充词是额外挑战。两者不能混在一起，否则孩子会觉得学习负担变重。**

---

## 12. Review Mechanism（错词复习机制）

第一版做轻量错词本。

### 12.1 错词进入规则

以下情况进入错词本：

- 第一次答错；
- 第二次仍答错；
- 挑战阶段答错。

记录内容：

- 单词 id；
- 所属 Unit；
- 错误次数；
- 最近错误时间；
- 是否已掌握。

### 12.2 错词复习规则

每个 Unit 主线关卡完成后，生成：

```text
Unit 错词复习关
```

错词复习仍然使用选择题。

不要求拼写。

### 12.3 已掌握规则

错词在复习中答对后，可以标记为：

```text
已掌握
```

第一版不做复杂间隔重复算法。

不做：

- 艾宾浩斯曲线；
- 多日复习计划；
- 自动推送复习；
- 复杂记忆强度评分。

## 13. Parent Panel（家长面板）

第一版做轻量家长面板。

家长面板要回答三个问题：

1. 孩子学到哪里了？
2. 哪些词还不会？
3. 最近使用效果怎么样？

### 13.1 家长面板展示内容

包括：

- Unit 完成进度；
- 每个关卡星级；
- 错词列表；
- 已学单词数；
- 已完成关卡数；
- 当前错词数量；
- Unit 状态：未开始 / 学习中 / 主线完成 / 完全掌握。

### 13.2 家长面板不做的内容

第一版不做：

- 家长账号；
- 孩子账号；
- 长期趋势图；
- 周报；
- 月报；
- 词表编辑后台；
- 学习计划设置；
- 多孩子管理。

## 15. Technical Constraints（技术约束）

### 15.1 数据管理

第一版使用：

```text
本地 JSON 文件
```

不做数据库。

每个单词建议包含以下字段：

```json
{
  "id": "u1-a-001",
  "unitId": "unit-1",
  "unitTitle": "Our Animal Friends",
  "levelType": "main",
  "levelId": "u1-main-1",
  "word": "animal",
  "meaning": "动物",
  "tags": ["animal"],
  "priority": "A",
  "image": "🐾",
  "audio": ""
}
```

字段说明：

- `id`：单词唯一 ID；
- `unitId`：所属 Unit；
- `unitTitle`：Unit 名称；
- `levelType`：`main` 或 `extra`；
- `levelId`：所属关卡；
- `word`：英文；
- `meaning`：中文；
- `tags`：主题标签；
- `priority`：A 类重点词 / B 类补充词；
- `image`：emoji、图标或图片路径；
- `audio`：音频路径，第一版可为空。

### 15.2 媒体资源

第一版使用：

- emoji；
- 简单图标；
- 占位图片；
- 浏览器 Text-to-Speech 发音。

第一版不接入：

- 第三方词典；
- AI 图片；
- AI 发音；
- 在线图片搜索。

### 15.3 进度保存

第一版不做登录。

使用浏览器本地保存进度。

需要保存：

- 已完成关卡；
- 每关星级；
- 错词记录；
- 已掌握错词；
- Unit 状态；
- Unit 徽章状态；
- 基础统计。

限制：

- 换设备不同步；
- 换浏览器不同步；
- 清除浏览器数据后进度可能丢失。

### 15.4 设备适配

优先适配：

- 电脑；
- 平板。

手机做基础适配，但不是主体验中心。

界面要求：

- 大按钮；
- 大字号；
- 点击区域足够大；
- 题目和选项清楚；
- 适合 8 岁孩子独立点击。

### 15.5 视觉风格

采用：

```text
清爽轻游戏风
```

要求：

- 明亮；
- 简洁；
- 亲和；
- 大按钮；
- 小动画；
- 不做复杂背景；
- 不做重度儿童游戏 UI。

核心原则：

**视觉服务学习，不抢走学习注意力。**

---

## 16. Learning Rhythm（学习节奏）

每日推荐：

```text
5–10 分钟
```

每日最低目标：

```text
完成 1 个小关，也就是 5 个单词
```

完成后可以继续下一关，但不强制。

第一版不做：

- 强制打卡；
- 连续签到压力；
- 排行榜；
- 学习时长惩罚；
- 每日任务轰炸。

目标不是让孩子一次学很多，而是：

**每天打开，完成一小关，不讨厌，第二天还愿意回来。**

## 18. Success Criteria（成功标准）

第一版成功，不看功能数量，看核心闭环有没有跑通。

### 18.1 孩子愿意使用

一周内，孩子至少有 4 天愿意打开网页。

这是最高优先级。

如果孩子不愿意打开，其他都没意义。

### 18.2 能完成每日最低目标

孩子每天能完成至少 1 个小关。

也就是：

```text
5 个单词
+
学习阶段
+
练习阶段
+
挑战阶段
+
星级反馈
```

### 18.3 Unit 1 能完整闭环

MVP 阶段，Unit 1 必须完整跑通：

- 主线关卡完成；
- 错词被记录；
- 错词复习完成；
- 额外挑战完成；
- 特殊星星获得；
- Unit 徽章解锁；
- 家长面板能看到结果。

### 18.4 错词复习有效

孩子第一次答错的词，在 Unit 错词复习中正确率有所提高。

不要求一次全部记住。

但要看到：

- 错词数量减少；
- 重复错误减少；
- 听音选义能力提高；
- 看图选英文能力提高。

### 18.5 家长能看懂学习情况

家长通过面板能清楚知道：

- 学到了哪个 Unit；
- 完成了几个关卡；
- 获得了多少星；
- 哪些词还容易错；
- 当前错词数量；
- Unit 是否主线完成；
- Unit 是否完全掌握。

### 18.6 技术稳定可用

第一版需要在电脑和平板上稳定运行。

至少做到：

- 页面能正常打开；
- Unit 1 能进入；
- 学习阶段能展示；
- 选择题能答；
- 发音能播放；
- 星级能保存；
- 错词能记录；
- 错词复习能完成；
- 徽章能解锁；
- 家长面板能显示；
- 刷新页面后进度不丢失。

## 19. MVP Scope（一句话版）

第一版要做的是：

**一个面向 8 岁三年级孩子的闽教版三年级下册英语单词轻游戏化学习网页。数据层先整理 Unit 1–4，但功能层先完整打穿 Unit 1。Unit 1 以课本重点词为主线，每关 5 个词，通过学习、练习、挑战三段式训练孩子认识和听懂单词；答错词进入错词本，主线完成后做错词复习；补充词作为额外挑战，完成后获得特殊星星和 Unit 徽章；进度保存在本地，家长可查看基础学习结果。**

## 20. Build Priority（开发优先级）

### P0：词库与数据结构

先完成：

1. Unit 1 A 类重点词；
2. Unit 1 B 类补充词；
3. Unit 1 关卡拆分；
4. JSON 数据结构；
5. emoji / 图标占位。

### P1：核心学习流程

完成：

1. 首页；
2. Unit 列表页；
3. Unit 1 关卡页；
4. 学习阶段；
5. 练习阶段；
6. 挑战阶段；
7. 发音播放；
8. 答题反馈。

### P2：学习闭环

完成：

1. 星级评分；
2. 错词记录；
3. 错词复习；
4. 主线完成状态；
5. 额外挑战关；
6. 特殊星星；
7. Unit 徽章。

### P3：家长面板与本地保存

完成：

1. 本地进度保存；
2. 家长面板；
3. 基础统计；
4. 刷新后数据保留；
5. 重置进度按钮。

### P4：Unit 2–4 扩展

在 Unit 1 被孩子真实使用后，再扩展：

```text
Unit 2 My Clothes
Unit 3 Time
Unit 4 Healthy Eating
```

不要在 Unit 1 验证前急着扩展。

## 21. Final Judgment（战略判断）

这版 SPEC 已经够用。

现在最大的敌人不是技术难度，而是**继续加功能的冲动**。

不要现在做账号。  
不要现在做金币。  
不要现在做宠物。  
不要现在做完整后台。  
不要现在做全册完整体验。  

真正要验证的是：

```text
孩子会不会愿意每天打开？
孩子能不能完成 5 个词？
孩子答错后会不会愿意再试？
错词复习有没有效果？
家长能不能看懂结果？
```

只要 Unit 1 跑通，这个项目就有继续做的价值。  
如果 Unit 1 跑不通，做再多 Unit 都只是扩大错误。
